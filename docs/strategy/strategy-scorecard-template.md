## 3. 수익성 지표 (Performance Metrics)

> 단일 자산(BTC/USDT) 기준 표본 내(IS) 결과를 기입한다.
> 각 지표의 **초보자 설명**, **계산 방법**, **판정 기준**을 함께 기록한다.

### 3.1 수익률 지표

| 지표 | 값 | 판정 |
|------|---|------|
| **Total Return (총 수익률)** | % | |
| **CAGR (연복리 수익률)** | % | |

> - **Total Return**: 투자 시작부터 끝까지의 누적 수익률. 예: 100만원 → 150만원이면 +50%.
> - **CAGR**: 매년 일정 비율로 성장했다고 가정한 연평균 수익률. Total Return을 연 단위로 환산한 것.
>   - 계산: `(최종자산/초기자산)^(1/년수) - 1`
>   - 판정: 양수이면 기본 충족, BTC Buy&Hold CAGR 대비 비교

### 3.2 위험 조정 지표

| 지표 | 값 | 판정 기준 | 판정 |
|------|---|----------|------|
| **Sharpe Ratio (샤프 비율)** | | > 1.0 우수, > 2.0 탁월 | |
| **Sortino Ratio (소르티노 비율)** | | > 1.5 우수 | |
| **Calmar Ratio (칼마 비율)** | | > 1.0 우수 | |

> - **Sharpe Ratio**: 위험 1단위를 감수했을 때 얼마나 초과 수익을 얻는가.
>   - 계산: `(전략수익률 - 무위험이자율) / 수익률표준편차`
>   - 1.0 미만이면 위험 대비 보상이 부족하다. 3.0 이상이면 오류를 의심한다.
> - **Sortino Ratio**: Sharpe와 유사하지만, **하방 변동성(손실)**만 고려한다.
>   - 이익이 크게 나는 것은 위험이 아니므로, 하방 리스크만 측정하는 것이 더 합리적이다.
> - **Calmar Ratio**: `CAGR / 최대낙폭(MDD)`. 최악의 손실 대비 연간 수익이 충분한지 판단한다.

### 3.3 낙폭(Drawdown) 지표

| 지표 | 값 | 판정 기준 | 판정 |
|------|---|----------|------|
| **Max Drawdown (MDD, 최대 낙폭)** | % | < 25% 우수, < 40% 허용 | |
| **Avg Drawdown (평균 낙폭)** | % | | |
| **MDD Duration (최대 낙폭 지속기간)** | 일 | < 180일 | |

> - **Max Drawdown**: 고점에서 저점까지 최대 하락폭. 투자자가 겪을 수 있는 **최악의 손실**.
>   - 예: 100만원 → 75만원이면 MDD = 25%.
>   - 40%를 초과하면 전략 폐기를 고려한다 (1단계 기준).
> - **Avg Drawdown**: 모든 하락 구간의 평균 하락폭. 일상적으로 겪는 손실 수준.
> - **MDD Duration**: 고점 회복까지 걸린 최장 기간(일). 길수록 심리적 압박이 크다.

### 3.4 거래 통계

| 지표 | 값 | 판정 기준 | 판정 |
|------|---|----------|------|
| **Total Trades (총 거래)** | | > 50 충분, > 100 우수 | |
| **Trades/Year (연간 거래)** | | | |
| **Win Rate (승률)** | % | > 45% (추세추종), > 55% (평균회귀) | |
| **Profit Factor (수익 팩터)** | | > 1.3 우수 | |
| **Avg Win (평균 수익)** | % | | |
| **Avg Loss (평균 손실)** | % | | |
| **Risk-Reward Ratio** | | > 1.5 우수 | |
| **Winning Trades** | | | |
| **Losing Trades** | | | |
| **Avg Trade Duration** | 시간 | | |

> - **Win Rate**: 전체 거래 중 수익 거래의 비율. 추세추종은 40-50%, 평균회귀는 55-65%가 일반적.
> - **Profit Factor**: `총이익 / 총손실`. 1.0이면 본전, 1.3 이상이면 수수료 차감 후에도 수익.
> - **Risk-Reward Ratio**: `평균수익 / 평균손실`. 승률이 낮아도 이 비율이 높으면 수익이 가능.
>   - 예: 승률 40%, Risk-Reward 2.0이면 → 기대값 = 0.4×2 - 0.6×1 = +0.2 (양수)

### 3.5 변동성 및 분포

| 지표 | 값 | 판정 기준 | 판정 |
|------|---|----------|------|
| **Volatility (연환산 변동성)** | % | < CAGR이 이상적 | |
| **Skewness (왜도)** | | > 0 (우측 꼬리) 선호 | |
| **Kurtosis (첨도)** | | < 5 안정, > 10 극단적 | |

> - **Volatility**: 수익률의 표준편차를 연 단위로 환산. 수익률이 얼마나 널뛰는지 측정.
> - **Skewness**: 수익률 분포의 비대칭성.
>   - 양수(>0): 큰 수익이 간헐적으로 발생 (추세추종에 유리)
>   - 음수(<0): 큰 손실이 간헐적으로 발생 (위험)
> - **Kurtosis**: 분포의 꼬리 두께. 3이면 정규분포, 높을수록 극단적 이벤트가 자주 발생.
>   - 암호화폐는 일반적으로 5-15 범위 (Fat tail).

## 4. 알파/베타 분석

> 벤치마크(BTC Buy & Hold) 대비 전략의 부가가치를 분석한다.

| 지표 | 값 | 설명 |
|------|---|------|
| **Benchmark Return** | % | BTC Buy & Hold 수익률 |
| **Alpha (초과 수익)** | % | 전략 수익 - 벤치마크 수익 |
| **Beta (시장 민감도)** | | 1.0=시장과 동일, 0=무관, <0=반대 |
| **Correlation (상관계수)** | | -1~+1, 0에 가까울수록 분산 효과 큼 |
| **Information Ratio** | | Alpha / Tracking Error (위험조정 알파) |
| **Tracking Error** | % | 벤치마크와의 수익률 편차(표준편차) |

> - **Alpha**: 시장(벤치마크)을 이긴 순수 초과 수익. 양수면 전략이 가치를 더한 것.
>   - 주의: Alpha가 크더라도 Beta가 높으면 시장 상승에 편승한 것일 수 있다.
> - **Beta**: 시장이 10% 오를 때 전략이 몇 % 움직이는가.
>   - Beta=0.5이면 시장의 절반만 따라간다 → 방어적 전략.
>   - Beta < 0이면 시장 하락 시 수익 → 헤지 전략.
> - **Information Ratio**: 벤치마크 대비 추가 위험 1단위당 초과 수익. > 0.5이면 우수.

### 4.1 요인 분해 (Factor Decomposition)

| 요인 | 노출도 | 설명 |
|------|-------|------|
| **Momentum** | | 모멘텀 요인 노출 |
| **Volatility** | | 변동성 프리미엄 노출 |
| **Liquidity** | | 유동성 프리미엄 노출 |
| **Residual Alpha** | | 요인 조정 후 순수 알파 |

> **Residual Alpha**: 알려진 요인(모멘텀, 변동성 등)으로 설명되지 않는 순수 초과 수익.
> 이것이 클수록 전략의 고유 가치가 높다.

## 5. 분포 분석 (Tail Risk)

> 극단적 시장 상황에서 전략이 어떻게 반응하는지 분석한다.

| 지표 | 값 | 판정 기준 | 판정 |
|------|---|----------|------|
| **Tail Ratio (꼬리 비율)** | | > 1.0 (우측꼬리 > 좌측꼬리) | |
| **VaR 95% (일일)** | % | | |
| **CVaR 95% (일일)** | % | | |

> - **Tail Ratio**: `95번째 백분위수 / 5번째 백분위수` (절대값).
>   - \> 1.0이면 큰 수익이 큰 손실보다 빈번하다 (좋은 신호).
> - **VaR (Value at Risk)**: "95% 확률로 하루 최대 이 정도까지 잃을 수 있다."
>   - 예: VaR 95% = -2%이면, 20거래일 중 1일은 2% 이상 손실 가능.
> - **CVaR (Conditional VaR)**: VaR를 초과하는 손실의 평균. "최악의 5% 날에 평균 얼마나 잃는가."
>   - CVaR는 항상 VaR보다 크다. 꼬리 위험의 심각도를 측정한다.

## 6. 비용 분석

> 거래 비용이 전략 수익에 미치는 영향을 분석한다.
> 비용을 고려하지 않으면 실전에서 적자가 될 수 있다.

### 6.1 비용 모델 설정

| 항목 | 값 | 설명 |
|------|---|------|
| **Maker Fee** | 0.02% | 지정가 주문 수수료 |
| **Taker Fee** | 0.04% | 시장가 주문 수수료 |
| **Slippage** | 0.05% | 호가 미끄러짐 |
| **Funding Rate (8h)** | 0.01% | 무기한 선물 펀딩비 |
| **Market Impact** | 0.02% | 시장 충격 |
| **편도 비용** | 0.11% | fee + slippage + impact |
| **왕복 비용** | 0.22% | 진입 + 청산 |

### 6.2 비용 영향

| 항목 | 값 |
|------|---|
| **비용 전 Sharpe** | |
| **비용 후 Sharpe** | |
| **Sharpe 감소율** | % |
| **연간 총 비용** | % |
| **손익분기 거래 수익률** | % (왕복 비용 이상이어야 수익) |
| **비용 민감도** | 10bps 추가 시 Sharpe 변화: |

> - **손익분기 거래 수익률**: 한 번의 거래(진입+청산)에서 최소 왕복 비용(0.22%) 이상의
>   수익을 내야 본전이다. 거래 빈도가 높을수록 이 부담이 누적된다.
> - **비용 민감도**: 비용을 10bps(0.1%) 추가했을 때 Sharpe가 얼마나 떨어지는가.
>   변화가 크면 "비용에 취약한 전략"이다.

## 7. 레짐별 성과

> 시장 상태(상승/하락/횡보)에 따라 전략 성과가 어떻게 달라지는지 분석한다.
> 특정 레짐에서만 수익이 나는 전략은 위험하다.

| 레짐 | Sharpe | CAGR | MDD | 기간 비율 |
|------|--------|------|-----|----------|
| **Bull (상승장)** | | % | % | % |
| **Bear (하락장)** | | % | % | % |
| **Sideways (횡보장)** | | % | % | % |

| 분석 항목 | 값 |
|----------|---|
| **최악 레짐** | |
| **최악 레짐 Sharpe** | |
| **레짐 전환 시 MDD** | % |

> - **레짐 분류 기준**: BTC 200일 이동평균 기준. 위=Bull, 아래=Bear, ±5% 이내=Sideways.
> - **레짐 전환 시 MDD**: 상승→하락 또는 하락→상승 전환 구간에서의 최대 낙폭.
>   추세추종 전략은 레짐 전환 시 whipsaw(횡보 손실)에 취약하다.

## 8. 전략 개발 파이프라인 체크리스트 (Stage 0-7)

> MC Coin Bot의 전략 개발 파이프라인 각 단계별 통과/탈락 판정.
> 참조: [퀀트 전략 발굴부터 실전 배포까지: 완전 가이드](docs/research/strategy-development-tuning-guide-2026.md)

### Gate 0: 아이디어 발굴

| 항목 | 값 |
|------|---|
| **판정** | `PASS` / `FAIL` |
| **평가표 점수** | /30 |
| **경제적 논거** | |
| **날짜** | |

### Gate 1: 시제품 백테스트 (표본 내)

| 항목 | 값 | 기준 |
|------|---|------|
| **판정** | `PASS` / `FAIL` / `WATCH` | |
| **IS Sharpe** | | > 1.0 |
| **Total Return** | % | > 10% |
| **Total Trades** | | > 50 |
| **MDD** | % | < 40% |
| **Profit Factor** | | > 1.0 |
| **날짜** | | |

> 즉시 폐기: Sharpe < 0.5 + Trades < 20, 또는 총수익 < 0%

### Gate 2: 표본 내/외 검증 (IS/OOS)

| 항목 | 값 | 기준 |
|------|---|------|
| **판정** | `PASS` / `FAIL` | |
| **IS Sharpe** | | |
| **OOS Sharpe** | | > 0.5 |
| **감쇠율** | % | < 50% |
| **OOS Total Return** | % | > 0% |
| **OOS Trades** | | > 10 |
| **날짜** | | |

> 감쇠율 = `(IS Sharpe - OOS Sharpe) / IS Sharpe × 100%`
> 50% 이상이면 과적합 의심.

### Gate 3: 파라미터 조율

| 항목 | 값 | 기준 |
|------|---|------|
| **판정** | `PASS` / `FAIL` | |
| **파라미터 고원 존재** | Y / N | 고원이 있어야 PASS |
| **±20% 변동 안정성** | Y / N | 파라미터 ±20% 변경 시 Sharpe 부호 유지 |
| **최적 파라미터** | | |
| **날짜** | | |

> **파라미터 고원 (Plateau)**: 특정 파라미터 값 하나에서만 수익이 나는 것이 아니라,
> 넓은 범위에서 안정적으로 수익이 나는 "평탄한 영역"이 존재하는가.
> 뾰족한 봉우리(spike)만 있으면 과적합이다.

### Gate 4: 심층 검증 (WFA / CPCV / PBO / DSR)

| 항목 | 값 | 기준 |
|------|---|------|
| **판정** | `PASS` / `FAIL` | |
| **WFE (Walk-Forward Efficiency)** | % | > 50% |
| **PBO (Prob. of Backtest Overfitting)** | % | < 40% |
| **DSR (Deflated Sharpe Ratio)** | | > 0.95 |
| **CPCV Sharpe** | | > 0 |
| **날짜** | | |

> - **WFE**: `(OOS 연수익률 / IS 연수익률) × 100%`. 학습 구간의 성과가 실전에서 얼마나 유지되는가.
> - **PBO**: 무작위 파라미터 선택이 우연히 좋은 결과를 낼 확률. 낮을수록 좋다.
> - **DSR**: 여러 번 시도한 것을 보정한 Sharpe. "이 Sharpe가 운이 아닐 확률."
> - **CPCV**: Combinatorial Purged Cross-Validation. 모든 가능한 IS/OOS 조합을 테스트.

### Gate 5: 최적 자산 선정

> 티어 1 자산 후보군을 **각각 개별 백테스트**하여 최고 성과 자산을 선택한다.
> 백테스트 기간은 반드시 **상승장·하락장·횡보장이 모두 포함**된 기간이어야 한다 (최소 3년).

| 자산 | Sharpe | CAGR | MDD | Total Trades | 비고 |
|------|--------|------|-----|-------------|------|
| | | % | % | | |

| 항목 | 값 | 기준 |
|------|---|------|
| **판정** | `PASS` / `FAIL` | |
| **백테스트 기간** | | 상승/하락/횡보 레짐 모두 포함 필수 |
| **최적 자산** | | Sharpe 최고 + MDD 허용 범위 내 |
| **차선 자산** | | 2순위 (대체 후보) |
| **날짜** | | |

> - **선정 기준**: Sharpe가 가장 높으면서 MDD < 40%인 자산 선택.
> - **기간 요건**: 최소 3년 이상, 상승·하락·횡보 레짐이 모두 포함되어야 백테스트 신뢰성 확보.
> - **왜 개별 자산인가**: 멀티에셋 포트폴리오보다 구현이 단순하고 에러 확률이 낮다.

## 9. 2026년 트렌드 대비 평가

> 최신 퀀트 트렌드 관점에서 전략의 미래 경쟁력을 평가한다.

| 항목 | 값 | 설명 |
|------|---|------|
| **IC Score** | | Information Coefficient. 시그널이 미래 수익을 얼마나 예측하는가 (0~1) |
| **On-Chain Alignment** | Y / N | 온체인 지표(NVT, 고래 이동 등)와 정합하는가 |
| **Factor Model** | | CF Benchmarks 7요인 분해 적용 여부 |
| **Regime Detection** | Y / N | HMM 등 레짐 탐지와 연동하는가 |
| **Alpha Decay Estimate** | %/년 | 예상 알파 감쇠율 (크립토 평균: 5-10%/년) |
| **Crowding Risk** | 상 / 중 / 하 | 동일 전략을 사용하는 참가자 수 추정 |

> - **IC Score**: 0.05 이상이면 의미 있는 예측력.
> - **Alpha Decay**: 논문 발표 후 평균 63% Sharpe 하락. 비공개 전략일수록 감쇠가 느리다.
> - **Crowding Risk**: SSRN 발표 전략은 "상", 자체 개발은 "하".
