SuperTrend + ADX 전략의 백테스트 결과 및 개선 이력.
멀티 타임프레임 × 멀티 에셋 검증, 숏 모드, PM 방어, 분할 진입 실험 포함.

## Strategy Overview

| 항목 | 설명 |
|------|------|
| **전략명** | `supertrend` |
| **로직** | SuperTrend(ATR 10, mult 3.0) + ADX(14) >= 25 필터 |
| **포지션** | Long-only, 전체 진입 / 전체 청산 (레버리지 1x) |
| **최적 타임프레임** | **12H** (4TF×20에셋 검증 결과) |
| **파일** | `src/strategy/supertrend/` (config, preprocessor, signal, strategy) |
| **설정** | `config/supertrend_btc.yaml` |

### Signal Logic (현재 — v1.1 + Pyramid)

```text
[전봉 기준 — Shift(1) Rule 적용]

기본 모드 (전체 진입):
  Entry: SuperTrend(t-1) == 초록 AND ADX(t-1) >= 25
  Exit:  SuperTrend(t-1) == 빨강  OR  ADX(t-1) < 25

분할 진입 모드 (Pyramid):
  Stage 1 (40%): SuperTrend 초록 + ADX >= 25 → 시그널 발생
  Stage 2 (+35%): close(t-1) > 20봉 최고가 → 신고점 돌파 확인
  Stage 3 (+25%): ADX(t-1) >= 30 → 추세 강화 확인
  Exit: 전체 일괄 청산 (ST 빨강 OR ADX < 25)

진입: 보수적 (AND), 청산: 공격적 (OR)
```

### SuperTrend 지표 핵심

```text
upper_band = HL2 + 3.0 × ATR(10)
lower_band = HL2 - 3.0 × ATR(10)

Ratchet 메커니즘:
  - 상승추세: lower_band는 올라가기만 (트레일링 지지선)
  - 하락추세: upper_band는 내려가기만 (트레일링 저항선)

추세 전환:
  - 초록→빨강: close < lower_band (지지선 붕괴)
  - 빨강→초록: close > upper_band (저항선 돌파)
```

### 왜 PM 방어가 역효과인가?

SuperTrend **자체가 이미 Trailing Stop**:
- lower_band의 Ratchet 메커니즘 = 내장형 동적 손절
- 외부 SL 10% 추가 → **이중 손절** → 조기 청산 → 재진입 비용 증가
- SL 10%가 SuperTrend 시그널보다 먼저 발동 → 시그널 간섭

**결론: SuperTrend는 외부 PM 방어 불필요. 내장 손절로 충분.**

TSMOM/Donchian은 내장 손절이 없어 PM 방어가 효과적이지만,
SuperTrend는 지표 자체에 손절이 내장되어 있어 구조적으로 다름.

### 왜 Pyramid이 효과적인가?

1. **초기 노출 제한**: 40%만 진입 → 하락 시 손실 40%로 제한 (외부 SL 불필요)

4. **승률 대폭 개선**: 소규모 초기 진입 → 실패 시 작은 손실 → WR 53.7% (Full 36%)

**핵심**: 외부 PM 방어(SL/TS)는 SuperTrend 내장 손절과 **간섭**하여 역효과.
분할 진입은 **비간섭적**으로 리스크 관리 → MDD 16.8%p 개선 달성.
