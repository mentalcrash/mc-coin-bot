## 4. Sharpe Ratio 분해

### 유형 분류

| 유형 | Sharpe | CAGR | Vol | 해석 |
|------|--------|------|-----|------|
| 고수익+고변동 | 1.5 | +80% | 50%+ | 강한 알파, 변동성 감내 필요 |
| 저수익+저변동 | 1.5 | +20% | 12% | 안정적이나 자본 효율 낮음 |
| 적정수익+적정변동 | 1.5 | +45% | 30% | **이상적** (vol-target 30-35%) |

### Red Flag: 비정상 Sharpe

| Sharpe | 해석 | 확인 사항 |
|--------|------|----------|
| > 3.0 | **과적합 강력 의심** | look-ahead bias, 비용 누락, 소수 거래 |
| > 2.5 | 과적합 의심 | G2 Decay 확인 필수 |
| 1.5-2.5 | 우수 (크립토 가능 범위) | 정상 범위 |
| 1.0-1.5 | 양호 | 배포 후보 |
| 0.5-1.0 | 보통 | 포트폴리오 분산용 |

## 5. 분석 프레임워크

### A. 단일 전략 분석

1. **수익성**: CAGR vs BTC B&H (알파 존재?), 비용 반영 전후 비교 (비용 > 수익의 30%이면 경고)
2. **리스크**: MDD 절대값 + 회복 기간, MDD/Vol-Target 비율, 최악 월간 수익률
3. **안정성**: 거래 수 충분성 (최소 30건), 수익의 시간 분포, 연도별 일관성
4. **과적합**: Sharpe > 2.0이면 "왜?", IS/OOS 갭 > 50%이면 과적합, DSR 결과

### B. 파라미터 스윕 분석

1. **Plateau**: 최적값 주변 ±20%에서 Sharpe 변동 < 10%이면 Robust. Sharp peak이면 과적합
2. **ShortMode**: DISABLED vs HEDGE_ONLY vs FULL — HEDGE_ONLY가 보통 최적
3. **최종 선정**: 최적값이 아니라 "robust 영역의 중심" 선택

### C. 결정 트리

```
Sharpe > 0.5? → No → 폐기 또는 근본 수정
  → Yes → 거래 수 > 30? → No → 데이터 기간 확장
    → Yes → IS/OOS 갭 < 50%? → No → 과적합, 파라미터 단순화
      → Yes → MDD < 30%? → No → 리스크 파라미터 강화
        → Yes → 비용 반영 후 양수? → No → 거래 빈도 감소
          → Yes → 배포 후보 → Paper Trading → Live
```

## 6. IS/OOS Decay 패턴 (Gate 2)

### Decay 해석 표

| Decay | 등급 | 해석 | CTREND 비교 |
|-------|------|------|------------|
| < 20% | **우수** | 강한 일반화, 과적합 위험 낮음 | — |
| 20-35% | 양호 | 적절한 IS/OOS 갭 | CTREND: 33.7% |
| 35-50% | **경계** | 파라미터 의존성 주의 | 기준 상한 |
| > 50% | **과적합** | FAIL — 모델 단순화 필요 | — |

### Decay 음수 (OOS > IS) 해석

| 원인 | 가능성 | 확인 방법 |
|------|--------|----------|
| OOS 기간이 강세장 | 높음 | 레짐 달력 대조 |
| IS에 2022 약세장 포함 | 높음 | 데이터 분할 확인 |
| 코드 버그 | 낮음 | shift(1) 재검증 |

### G1 → G2 일관성

```
기대: G2 OOS Sharpe >= G1 Best Sharpe x 0.3
CTREND 실제: 1.78 / 2.05 = 87% 유지 (우수)
```

| **넓은 고원** | `___/‾‾‾‾‾‾\___` | **최적** — 로버스트 | CTREND alpha (0.1~1.0) |

### vol_target 특수 패턴

vol_target은 Sharpe에 거의 영향 없는 것이 **정상** (순수 레버리지 스케일링).
vol_target 변경 시 Sharpe도 크게 변하면 → 비선형 리스크 존재 (경고).

### 파라미터 간 교호작용 징후

| 징후 | 해석 |
|------|------|
| 파라미터 A 최적값이 B 값에 따라 이동 | 강한 교호작용 — 2D 스윕 필요 |
| 두 파라미터 모두 같은 방향에서 좋음 | 복합 효과 — 과적합 위험 |
| 각 파라미터가 독립적으로 고원 유지 | **이상적** — one-at-a-time 스윕 신뢰 가능 |

## 8. PBO 해석 (Gate 4)

### PBO 이중 경로 판정

| 경로 | 조건 | 설명 |
|:----:|------|------|
| **A** | PBO < 40% | 기본 경로 — 과적합 위험 낮음 |
| **B** | PBO < 80% AND CPCV 전 fold OOS > 0 AND MC p < 0.05 | 보조 경로 — 기저 alpha 견고 |

### PBO + CPCV 복합 해석

| PBO | CPCV 전 fold OOS | 판정 | 근거 |
|-----|:-----------------:|------|------|
| < 40% | 전부 양수 | **강건** (경로 A) | 과적합 위험 낮고 전 fold 수익 |
| < 40% | 일부 음수 | 양호 (경로 A) | 과적합 낮으나 약한 fold 주시 |
| 40-80% | **전부 양수** + MC p<0.05 | **PASS** (경로 B) | 기저 alpha 견고 |
| 40-80% | 일부 음수 | **FAIL** | 과적합 + 일부 fold 손실 |
| >= 80% | 무관 | **FAIL** | 경로 B 상한 초과 |

### 경로 B 적용 사례

- **CTREND** (PBO 60%): 전 fold OOS 양수, MC p=0.000 → 경로 B PASS
- **Anchor-Mom** (PBO 80%): 전 fold OOS 양수, MC p=0.000 → 경로 B PASS (상한 충족)

> PBO는 "순위 보존"을 측정하지만, 실전에서는 "어떤 파라미터를 골라도 수익이 나는가" (CPCV robustness)가 중요.

## 9. DSR n_trials 민감도 (Gate 4)

| 구분 | n_trials | 의미 | CTREND |
|------|----------|------|--------|
| batch | 동일 배치 전략 수 (5-10) | 해당 연구 세션 내 다중 비교 보정 | 7 → DSR 1.00 |
| all | 전체 시도 전략 수 (20-50) | 프로젝트 전체 다중 비교 보정 | 31 → DSR 0.16 |

| batch DSR | all DSR | 해석 |
|-----------|---------|------|
| > 0.95 | > 0.95 | **강건** — 전 프로젝트에서도 유의 |
| > 0.95 | < 0.95 | 양호 — 배치 내 유의, 전체 불확실 |
| < 0.95 | 무관 | **경고** — 배치 내에서도 불확실 |

> batch DSR > 0.95이 Gate 4 판정 기준. all DSR은 참고용.

## 12. Gate 간 일관성 패턴

### 건강한 패턴

```
G1 Sharpe 1.5 → G2 OOS 1.0 (Decay 33%) → G4 WFA OOS 0.8 (Decay 47%)
점진적 감소: 각 단계에서 20-40% 감쇠 = 정상적 일반화 손실
```

### 비건강한 패턴

- G1 2.5 → G2 0.3 (Decay 88%) → G4 0.1: 과적합 전형
- G1 1.0 → G2 1.5 (Decay -50%) → G4 0.2: 데이터 분할 인공물

### 일관성 체크 표

| 비교 | 건강한 범위 | 경고 조건 |
|------|-----------|----------|
| G2 OOS / G1 Sharpe | 0.3 ~ 1.0 | < 0.3 (과적합) 또는 > 1.0 (의심) |
| G4 WFA OOS / G2 OOS | 0.5 ~ 1.2 | < 0.5 (window 의존) |
| G4 Decay - G2 Decay | -10% ~ +15% | 차이 > 20%p (방법론 민감) |
| G3 baseline vs G1 Sharpe | 유사 (+-10%) | 차이 > 20% (파라미터 변경) |
